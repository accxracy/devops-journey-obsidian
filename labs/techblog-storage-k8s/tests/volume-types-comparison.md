# Сравнение типов Volumes (emptyDir, hostPath, PVC)


## 1. emptyDir (Временный диск)
* **Где использовали:** Кэш веб-сервера `blog-app` (`/tmp/cache`)
* **Поведение при удалении Pod'а:** Все данные **УДАЛЯЮТСЯ**
* **Для чего нужен:** Только для временных файлов. Жизненный цикл диска жестко привязан к жизни конкретного Pod'а

## 2. hostPath (Папка на ноде)
* **Где использовали:** Логи веб-сервера `blog-app` (`/var/log/app`)
* **Поведение при удалении Pod'а:** Данные **СОХРАНЯЮТСЯ**, так как файл физически записан на жесткий диск ноды
* **Минус:** Если кластер состоит из нескольких серверов (нод) и Pod после перезапуска переедет на другой сервер, он потеряет доступ к этой папке

## 3. PersistentVolumeClaim / PVC (Полноценный диск)
* **Где использовали:** База данных PostgreSQL и папка с картинками (`uploads-pvc`)
* **Поведение при удалении Pod'а:** Данные **СОХРАНЯЮТСЯ**
* **Главный плюс:** Это независимый жесткий диск. Если Pod упадет, Кубернетес аккуратно отсоединит этот диск и подключит его к новому Pod'у, где бы тот ни запустился. Это единственный безопасный способ хранить данные баз данных
