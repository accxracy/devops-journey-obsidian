# 🔐 Nginx: SSL Сертификаты и HTTPS (Certbot)

В современном вебе передавать данные по открытому HTTP (порт 80) недопустимо — провайдеры и злоумышленники могут перехватить пароли и токены. Трафик должен быть зашифрован через TLS/SSL (HTTPS, порт 443).

## 🛑 SSL Termination (Терминация SSL)
Это главный архитектурный паттерн DevOps для работы с шифрованием.
Тебе **не нужно** настраивать HTTPS внутри твоего Python, Go или Node.js приложения. Твое приложение внутри Docker-контейнера должно продолжать работать по обычному HTTP.

**Как это работает:**
1. Nginx принимает зашифрованный трафик из интернета по порту 443.
2. Nginx берет на себя всю вычислительную нагрузку по **расшифровке** этого трафика (Это и называется Терминацией).
3. Уже чистый, расшифрованный трафик Nginx отправляет внутрь сервера (в Docker-контейнер) по локальному HTTP (например, на `127.0.0.1:3000`).

*Плюсы:* Приложению не нужно тратить CPU на криптографию. Все сертификаты лежат в одном месте (на уровне Nginx), а не размазаны по 10 разным контейнерам.

## 🤖 Let's Encrypt и Certbot
Раньше SSL-сертификаты нужно было покупать за деньги на 1 год и вручную прописывать в конфиги. Сейчас индустриальным стандартом является **Let's Encrypt** — центр сертификации, который выдает сертификаты **бесплатно**, но сроком на 90 дней.

Чтобы не обновлять их руками каждые 3 месяца, используется утилита **Certbot**.

### Как настроить HTTPS (Алгоритм)
1. У тебя уже должен быть рабочий конфиг Nginx для твоего домена на порту 80 (базовый Reverse Proxy). Домен в DNS должен указывать на IP твоего сервера.
2. Устанавливаешь Certbot (в Ubuntu/Debian):
 `apt install certbot python3-certbot-nginx`
3. Запускаешь автоматическую настройку:
 `certbot --nginx -d example.com -d www.example.com`

### Что делает Certbot под капотом:
1. Обращается к серверам Let's Encrypt и доказывает, что ты физически владеешь сервером, на который указывает домен (ACME Challenge).
2. Скачивает приватный ключ (`privkey.pem`) и публичный сертификат (`fullchain.pem`) в папку `/etc/letsencrypt/live/example.com/`.
3. **Автоматически редактирует твой конфиг Nginx:**
 * Добавляет блок `listen 443 ssl`.
 * Прописывает пути к скачанным сертификатам.
 * Настраивает 301 редирект (чтобы пользователи, которые пишут `http://example.com`, принудительно перенаправлялись на защищенный `https://`).
4. Мягко перезапускает Nginx (`reload`).

## 🔄 Автообновление (Auto-renewal)
Поскольку сертификаты живут 90 дней, Certbot при установке автоматически создает системный таймер в Linux (через `systemd` или `cron`). 
Он запускается дважды в день и проверяет, не истекает ли сертификат. Если до конца осталось меньше 30 дней, Certbot сам скачивает новый сертификат и перезагружает Nginx.

**Команда для проверки (имитация обновления без реального скачивания):**
 `certbot renew --dry-run`

Если команда отработала без ошибок (Success) — про сертификаты можно забыть, они будут обновляться в фоне вечно.
