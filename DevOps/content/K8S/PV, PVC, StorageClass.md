# 💾 Kubernetes: Хранение данных (PV, PVC, StorageClass)

В Docker мы использовали `Volumes`, чтобы привязать папку на сервере к контейнеру. В Kubernetes так делать нельзя! 
**Причина:** Если сервер (Worker Node 1) сгорит, Кубернетес перезапустит твой Pod с базой данных на сервере (Worker Node 2). Но папки с данными на втором сервере нет! Данные потеряны навсегда.

Чтобы решить эту проблему, K8s отрывает хранилище данных от конкретного сервера. Диски живут отдельно (обычно в облаке — AWS EBS, Yandex Cloud Block Storage или на сетевом хранилище NFS), а Поды подключаются к ним по сети.

Для этого используется связка из 3 абстракций:

## 💽 1. PersistentVolume (PV) — «Физический диск»
**PV** — это реальный кусок хранилища (жесткий диск) где-то в дата-центре. 
* Это ресурс **всего кластера**, он не принадлежит ни одному пространству имен (Namespace).
* Его создает администратор кластера (или облачный провайдер автоматически).
* *Пример в жизни:* Админ купил жесткий диск на 100 ГБ и воткнул его в кластер.

## 📄 2. PersistentVolumeClaim (PVC) — «Заявка на диск»
**PVC** — это запрос разработчика (тебя) на выделение места. 
* Ты пишешь манифест: *"Мне нужен диск на 10 ГБ, быстрый (SSD), чтобы моя БД могла в него писать"*.
* Кубернетес берет твою заявку (PVC), идет в список всех доступных дисков (PV), находит подходящий кусок и намертво **связывает (Bound)** их друг с другом.
* Теперь твой Pod может использовать этот PVC как обычную папку.

## 🏭 3. StorageClass (SC) — «Фабрика дисков»
Раньше админам приходилось вручную создавать PV (диски) каждый раз, когда разрабу нужна была БД. Это долго.
**StorageClass** автоматизирует этот процесс (Dynamic Provisioning).
* Это интеграция с облачным провайдером (AWS, Yandex Cloud, Selectel).
* *Как работает:* Ты создаешь PVC (Заявку на 10 ГБ) и указываешь `storageClassName: ssd-fast`. Облачный провайдер видит заявку, **сам** на лету создает реальный SSD-диск на 10 ГБ в дата-центре, сам создает PV в кластере и отдает его тебе. Девопсу вообще не нужно трогать железо.

## 🛠 Архитектурная цепочка (Как это выглядит в коде)

**Шаг 1: Создаем заявку (PVC)**

 apiVersion: v1
 kind: PersistentVolumeClaim
 metadata:
 name: postgres-pvc
 spec:
 accessModes:
  - ReadWriteOnce  # Диск может быть примонтирован только к одному узлу
 resources:
  requests:
  storage: 10Gi  # Просим 10 Гигабайт

**Шаг 2: Подключаем PVC к Поду (или Deployment)**

 apiVersion: apps/v1
 kind: Deployment
 metadata:
 name: postgres
 spec:
 template:
  spec:
  containers:
   - name: postgres
   image: postgres:15
   volumeMounts:
    - name: db-data
    mountPath: /var/lib/postgresql/data # Куда монтировать внутри Пода
  volumes:
   - name: db-data
   persistentVolumeClaim:
    claimName: postgres-pvc # Указываем имя нашей заявки из Шага 1
