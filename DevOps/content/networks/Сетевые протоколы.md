# 🔌 Основные сетевые протоколы и их порты

**Протокол** — это просто набор правил (язык), по которому общаются две программы. 
**Порт** — это "дверь" на сервере, за которой сидит программа, понимающая этот протокол.

Девопс обязан знать дефолтные порты наизусть для настройки файрволов и проброса портов в Docker (`-p 8080:80`).

## 🌍 Веб-трафик (L7 - Application Layer)
* **HTTP (Port 80 / TCP):** Протокол передачи гипертекста (сайтов). Трафик идет в открытом текстовом виде.
* **HTTPS (Port 443 / TCP):** Тот же HTTP, но завернутый в криптографический туннель TLS/SSL. Весь трафик зашифрован.
 * *DevOps Контекст:* Балансировщики Nginx обычно слушают порты 80 и 443. Если на 80 порт приходит запрос, Nginx делает 301 редирект на 443 порт, чтобы принудительно включить шифрование.

## 🛠 Администрирование и Передача файлов
* **SSH (Port 22 / TCP):** Secure Shell. Защищенный протокол удаленного управления сервером через консоль. Полностью вытеснил старый и небезопасный Telnet (порт 23).
* **FTP (Port 21 / TCP):** File Transfer Protocol. Старый протокол для передачи файлов. В современном DevOps используется редко (трафик не зашифрован). Вместо него используют **SFTP** (работает поверх SSH на порту 22).

## ✉️ Электронная почта
* **SMTP (Port 25 или 465 / TCP):** Simple Mail Transfer Protocol. Протокол для **отправки** писем между серверами.
* **IMAP / POP3 (Ports 143, 993 / TCP):** Протоколы для **чтения** (скачивания) входящих писем почтовым клиентом.

## 📡 Инфраструктурные протоколы
* **DNS (Port 53 / UDP & TCP):** Domain Name System. "Телефонная книга" интернета. Превращает `google.com` в IP-адрес.
 * *Особенность:* В 99% случаев использует быстрый UDP. TCP используется только при передаче больших объемов данных (когда DNS-сервера синхронизируют базы между собой).
* **NTP (Port 123 / UDP):** Network Time Protocol. Синхронизация времени на сервере. Если на сервере сбилось время, у тебя отвалятся SSL-сертификаты и генерация JWT-токенов.

## 🗄 Порты Баз Данных (Не протоколы, но база для DevOps)
Эти порты **никогда** не должны торчать наружу в открытый интернет без жесткой необходимости и файрвола.
* **PostgreSQL:** Port `5432`
* **MySQL:** Port `3306`
* **Redis:** Port `6379`
* **MongoDB:** Port `27017`
