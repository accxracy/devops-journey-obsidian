# 🐳 Docker: Основы и Архитектура

**Docker** — это платформа для упаковки, доставки и запуска приложений в изолированных средах (контейнерах).
Главная решаемая проблема в IT: *"У меня на ноуте всё работает, а на сервере упало"*. Docker гарантирует, что среда запуска (ОС, библиотеки, версии языка) будет абсолютно идентична везде.

## 📦 Контейнеры vs Виртуальные машины (VM)
Это первый вопрос на любом собеседовании.

* **Виртуальная машина (VM):** Эмулирует физическое железо. Внутри каждой виртуалки стоит полноценная гостевая ОС (со своим ядром), которая съедает минимум 1-2 ГБ оперативной памяти просто на свое существование. Стартует долго (минуты).
* **Контейнер (Docker):** Виртуализирует только рабочую среду, а не железо. У контейнеров **нет своей ОС**. Все контейнеры напрямую используют одно ядро твоего Linux-сервера. Занимают мегабайты, стартуют за миллисекунды.

*Как достигается изоляция, если ядро одно?*
За счет двух функций ядра Linux:
1. **Namespaces:** Матрица для процесса. Контейнер думает, что он один на сервере (изоляция процессов, сетей, файловой системы).
2. **Cgroups:** Лимиты. Не дают одному контейнеру сожрать 100% CPU и RAM всего сервера.

## 🧱 Три кита Docker

1. **Dockerfile (Рецепт):** Текстовый файл с пошаговыми инструкциями (возьми Python 3.10 -> скачай библиотеки -> запусти main.py).
2. **Image / Образ (Шаблон):** Собранный по Докерфайлу неизменяемый (Read-Only) слепок файловой системы. Это как установочный `.iso` образ, только для твоего приложения.
3. **Container / Контейнер (Объект):** Запущенный процесс. Экземпляр Образа. Из одного неизменяемого Образа можно одновременно запустить 100 независимых Контейнеров. Контейнер добавляет поверх образа тонкий слой для записи данных (Read-Write).

## 🛠 Базовый жизненный цикл (CLI)
* `docker build -t my_bot .` : Собрать Образ из Докерфайла в текущей папке и назвать его `my_bot`.
* `docker run -d -p 80:80 my_bot` : Запустить Контейнер из этого образа в фоновом режиме (`-d`) и прокинуть трафик с 80 порта сервера на 80 порт контейнера (`-p`).
* `docker ps` : Вывести список живых контейнеров.
* `docker logs -f <ID>` : Читать поток вывода (логи) контейнера в реальном времени.
* `docker stop <ID>` : Мягко остановить контейнер (отправляет `SIGTERM`).
