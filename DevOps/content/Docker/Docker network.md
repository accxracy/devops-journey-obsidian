# 🌐 Сети в Docker (Networks)

По умолчанию контейнер рождается в полной сетевой изоляции (NET Namespace). У него свой виртуальный интерфейс и локальный IP. Чтобы контейнеры могли общаться друг с другом и с миром, используются сетевые драйверы.

## 🌉 1. Драйвер Bridge (Мост) — По умолчанию
Виртуальный коммутатор (`docker0`).
* **Как работает:** Контейнеры в одной сети `bridge` изолированы от остального хоста, но могут общаться между собой.
* **Магия DNS:** Если контейнеры запущены через Docker Compose (создается `user-defined bridge`), внутри поднимается встроенный DNS. Бот обращается к базе не по IP, а по имени сервиса: `postgres:5432`.
* **Проброс портов (Port Mapping):** Флаг `-p 8080:80`. Докер берет трафик с порта 8080 сервера и кидает в 80 порт контейнера через правила `iptables`.

## 🏠 2. Драйвер Host (Хост)
Контейнер разделяет сетевое пространство хост-сервера.
* **Как работает:** Проброс портов не нужен. Если контейнер слушает порт 80, он сразу открывается на самом VPS.
* **Плюсы:** Максимальная скорость (нет NAT-маршрутизации).
* **Минусы:** Небезопасно. Возможен конфликт портов (нельзя запустить два Nginx на 80 порту).

## 🚫 3. Драйвер None
Полная изоляция (только `localhost`). Используется для параноидальных задач (генерация крипто-ключей), где нужен 100% air-gap от интернета.
