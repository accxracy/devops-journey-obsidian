# 💾 Хранение данных в Docker

Контейнеры не имеют состояния (Stateless). Вся их внутренняя файловая система уничтожается при удалении контейнера. Чтобы сохранить базу данных, логи или загруженные юзерами картинки, нужно прокинуть папку с хост-сервера внутрь контейнера. 

Есть два способа это сделать:

## 📂 1. Bind Mounts (Проброс папок хоста)
Ты жестко привязываешь конкретную папку на твоем сервере к папке внутри контейнера.
* **Как работает:** Ты говоришь Докеру: *"Возьми папку `/home/grishanya/configs` на сервере и примонтируй ее в `/app/configs` внутри контейнера"*.
* **Плюсы:** Ты можешь легко открывать и редактировать эти файлы прямо на сервере через `nano` или VS Code.
* **Минусы:** Зависимость от путей на сервере. Если ты перенесешь проект на другой сервер, где нет папки `/home/grishanya/configs`, контейнер упадет с ошибкой. Часто возникают проблемы с **правами доступа** (разные UID юзеров на хосте и в контейнере).
* **Когда использовать:** Во время локальной разработки (чтобы код обновлялся в контейнере при сохранении) или для проброса конфигов (например, `nginx.conf`).

## 📦 2. Docker Volumes (Именованные тома) — Best Practice
Тома полностью управляются самим Докером. Ты не указываешь конкретный путь на сервере, ты просто придумываешь Тому имя (например, `pg_data`).
* **Как работает:** Докер сам создает скрытую папку глубоко в ядре системы (обычно в `/var/lib/docker/volumes/pg_data/_data`) и монтирует ее в контейнер. 
* **Плюсы:** Максимальная безопасность. Отсутствие проблем с правами доступа. Идеально работает при переносе между Linux, Windows и Mac. Легко бэкапить.
* **Когда использовать:** Всегда на Продакшене. Особенно для баз данных (PostgreSQL, Redis).

## 🛠 Как это выглядит на практике

В консоли (флаг `-v`):
* Bind Mount: `docker run -v /home/user/data:/var/lib/postgresql/data postgres`
* Volume: `docker run -v my_db_data:/var/lib/postgresql/data postgres` (указано просто имя `my_db_data` вместо пути).

В **Docker Compose** (пример правильной базы данных):

 services:
  db:
   image: postgres:15
   environment:
    POSTGRES_USER: admin
    POSTGRES_PASSWORD: 123
   volumes:
    - pg_data:/var/lib/postgresql/data # Прокидываем Volume в папку БД

 # Обязательно объявляем Volume в конце файла!
 volumes:
  pg_data:

## 🧹 Управление Томами (Команды)
* `docker volume ls` : Показать все тома на сервере.
* `docker volume rm <name>` : Удалить конкретный том (⚠️ уничтожит данные!).
* `docker system prune --volumes` : Жесткая очистка. Удалит все тома, которые в данный момент не прикреплены ни к одному запущенному контейнеру. (Опасно: можно случайно удалить выключенную базу).
