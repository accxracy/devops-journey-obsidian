# 🐚 Переменные окружения и Bash

**Bash** (Bourne Again Shell) — это командная оболочка. Программа, которая принимает твои команды текстом и передает их ядру Linux. 
У Bash есть своя память — переменные окружения (Environment Variables). Это пары `КЛЮЧ=ЗНАЧЕНИЕ`, которые хранят настройки для текущего сеанса пользователя.

## 🔑 Базовые команды
* `env` или `printenv` : вывести список всех текущих переменных.
* `echo $USER` : вывести значение конкретной переменной (знак `$` означает "взять значение переменной").
* `MY_VAR="hello"` : создать локальную переменную (доступна только в текущем окне терминала).
* `export MY_VAR="hello"` : создать **глобальную** переменную (будет доступна всем дочерним процессам, запущенным из этого терминала).

## 🛣 Главная переменная: `$PATH`
Это самая важная переменная в Linux. Она содержит список директорий, разделенных двоеточием `:`.

* **Как это работает:** Когда ты пишешь команду `python3`, ядро не сканирует весь жесткий диск. Оно по очереди заглядывает в папки, указанные в `$PATH` (обычно это `/usr/bin`, `/bin`, `/usr/local/bin`). Как только находит там исполняемый файл `python3` — запускает его. Если не находит — выдает знаменитую ошибку `command not found`.
* **Как добавить свою программу:** Если ты скачал бинарник (например, `kubectl`) в свою папку `/home/user/kube`, чтобы запускать его отовсюду без полного пути, нужно добавить папку в PATH:
 `export PATH="$PATH:/home/user/kube"`

## 💾 Как сохранить переменные навсегда
Команда `export` живет ровно до закрытия терминала. Чтобы переменная загружалась всегда:

1. **`~/.bashrc`**: Скрипт, который автоматически выполняется каждый раз, когда ты открываешь новое окно терминала. 
2. **`~/.bash_profile`** (или `~/.profile`): Выполняется один раз при логине (вводе пароля) в систему.
*Решение:* Открыть `nano ~/.bashrc`, дописать в конец `export DB_PASS="123"`, сохранить и выполнить `source ~/.bashrc` (перезагрузить конфиг).

### 🐳 DevOps Контекст (The Twelve-Factor App)
Одно из главных правил правильной архитектуры приложений (12-Factor App): **Конфигурация должна храниться в среде выполнения, а не в коде.**

Ты никогда не хардкодишь токен телеграм-бота или пароль от базы данных в `main.py`.
Вместо этого:
1. В коде ты пишешь: `token = os.getenv("BOT_TOKEN")`.
2. В Докере или Кубере ты прокидываешь эту переменную снаружи.
