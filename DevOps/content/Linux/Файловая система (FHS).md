# 📂 Файловая система Linux (FHS)

Главное правило: **Всё начинается от корня `/`**. В Linux нет дисков `C:\` или `D:\`. Разные физические диски просто "монтируются" (привязываются) к разным папкам внутри этого единого дерева. Другое важное правило: **Всё есть файл** (включая жесткие диски, порты и процессы).

## 🗺 Карта директорий (Где что лежит)

### ⚙️ Ядро и загрузка
* `/boot` — Файлы для запуска системы (ядро `vmlinuz`, загрузчик GRUB, `initramfs`). **DevOps-контекст:** Если эта папка забивается старыми ядрами на 100%, сервер не сможет обновиться или перезагрузиться.

### 🛠 Исполняемые файлы (Программы)
* `/bin` — Базовые утилиты для всех юзеров (`ls`, `cat`, `bash`).
* `/sbin` — Системные утилиты для админа/root (`fdisk`, `iptables`, `ss`).
* `/usr/bin` — Программы, установленные пользователем (`python3`, `docker`).

### 📝 Конфигурации (Настройки)
* `/etc` — **Святая святых DevOps.** Здесь лежат ВСЕ глобальные конфиги. 
 * `/etc/nginx/nginx.conf` — конфиг веб-сервера.
 * `/etc/systemd/system/` — файлы твоих демонов.
 * `/etc/passwd` — список пользователей системы.
 * *Важно:* В `/etc` никогда не лежат исполняемые файлы, только текстовые настройки.

### 🔄 Переменные данные и Логи
* `/var` — Данные, которые постоянно меняются в размере (Variable).
 * `/var/log` — **Журналы системы и приложений.** Твой главный инструмент при падении сервера.
 * `/var/lib/postgresql/` — Физическое хранилище файлов баз данных.
 * `/var/lib/docker/` — Здесь Docker хранит образы, контейнеры и тома.

### 👤 Пользователи
* `/home` — Личные папки пользователей (например, `/home/grishanya`).
* `/root` — Домашняя папка Суперпользователя (админа). Она лежит отдельно от остальных, чтобы даже если раздел `/home` сломается, админ мог зайти.

### 🗑 Временные файлы
* `/tmp` — Мусорка. Любое приложение может писать сюда временные файлы. **Фича:** Папка полностью и безвозвратно очищается при каждой перезагрузке сервера.

### 🔌 Устройства и процессы (Виртуальные ФС)
* `/dev` — Устройства (Devices). Жесткие диски (`/dev/sda`), терминалы, и знаменитая "черная дыра" `/dev/null`, куда скидывают ненужный вывод команд.
* `/proc` — Виртуальная папка. Файлов там на диске нет, они генерируются в оперативке. В `/proc` лежат папки с номерами `PID` всех запущенных процессов.
* `/sys` — Тоже виртуальная папка для взаимодействия с ядром (изменение лимитов, параметров сети).

## 🆔 Inodes (Индексные дескрипторы)

В Linux имя файла и его физические данные лежат отдельно. Звено, которое их связывает это **inode**.
* **Что это:** Это "паспорт" файла. Структура данных, хранящая метаданные: размер, права доступа, владельца, таймстемпы и, главное, ссылки на физические блоки жесткого диска, где лежат сами байты.
* **Чего НЕТ в inode:** Имени файла. Имя файла хранится в самой директории (директория в Linux — это просто текстовая таблица формата `"имя файла" -> "номер inode"`).
* ⚠️ **DevOps Баг "Исчерпание айнодов":** При форматировании диска (например, в `ext4`) создается фиксированное количество inodes (правило: 1 файл = 1 inode). Если приложение сойдет с ума и создаст миллионы файлов по 1 байту, дисковое пространство (`df -h`) останется свободным, но "паспорта" закончатся. Система выдаст ошибку `No space left on device`, и ты не сможешь создать даже пустой файл. 

**Полезные команды:**
`ls -i <file>` : посмотреть номер inode конкретного файла
`df -i` : проверить свободные/занятые inodes на смонтированных дисках

## 🔗 Ссылки (Links)

Учитывая архитектуру Inodes, в Linux существует два вида ярлыков:

1. **Hard Link (Жесткая ссылка):** 
  * Это просто еще одно "имя" в директории, которое указывает на **тот же самый inode**, что и оригинальный файл.
  * Если удалить оригинал (команда `rm` просто удаляет строчку с именем из директории), данные на диске останутся целы, и они будут доступны по жесткой ссылке.
  * *Ограничения:* Нельзя сделать для директорий. Нельзя сделать ссылку на файл, лежащий на другом жестком диске.

2. **Symlink (Символическая/Мягкая ссылка):** 
  * Классический ярлык. У него есть **свой собственный inode**, а внутри него текстом прописан путь к оригинальному файлу.
  * Если удалить оригинал — ссылка станет битой ("сиротой"), при попытке ее открыть будет ошибка.
  * *Команда создания:* `ln -s /path/to/original /path/to/link`
