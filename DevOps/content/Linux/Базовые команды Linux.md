# 🛠 Базовые команды Linux

## 🌐 Сети
`curl -I https://domain.com/` : запрос HTTP-заголовка
* `-I`: отправляет `HEAD` запрос, а не `GET` (быстро проверяет статус-код без скачивания страницы)
* `-v`: verbose-режим, показывает процесс DNS-резолва и установку соединения

`ss -tulnp` : список слушающих портов и процессов 
* `-t`: TCP порты
* `-u`: UDP порты
* `-l`: слушающие 
* `-p`: показать PID процесса

`lsof -i :<port>` : список процессов, которые занимают конкретный порт
* `-i`: фильтр только по сетевым соединениям

## 💾 Диски и файлы
`df -h` : процент заполненности смонтированных дисков
* `-h`: в понятном формате (human-readable: Mb, Gb)

`du -sh /*` : считает вес директорий в текущей директории
* `-s`: вывести общий размер папки (summarize)
* `-h`: в понятном формате (human-readable)

`tail -f <file>` : чтение логов
* `-f`: чтение в реальном времени (follow)

`grep -rnw <path> -e <pattern>` : поиск текста внутри файлов
* `-r`: рекурсивно по папкам
* `-n`: номер строки
* `-w`: слово целиком

`find <path> -name <pattern>` : поиск файлов по имени

## ⚙️ Процессы и система
`ps aux` : полный список процессов в системе

`kill <PID>` : завершение процесса (по умолчанию `SIGTERM`)
* `-9`: принудительное завершение процесса на уровне ядра (`SIGKILL`)

`journalctl -u <service> -f` : системные логи демонов ([[systemd]])
* `-u`: фильтр по конкретной службе
* `-f`: чтение логов в реальном времени (follow)

`systemctl status <service>` : состояние службы 
`systemctl restart <service>`: перезапуск службы
`systemctl enable <service>` : добавить службу в автозагрузку

`htop` / `top` : монитор ресурсов
`cat <file>`: вывести содержимое файла в консоль
`touch <file>` : создать файл

## 🐳 [[Docker]]
`docker compose logs -f --tail n <container>` : логи контейнера
* `-f`: чтение в реальном времени
* `--tail n`: показать только последние `n` строк

`docker exec -it <container> /bin/bash` : провалиться в консоль контейнера
* `-i`: оставляет поток ввода открытым 
* `-t`: выделяет псевдотерминал для ввода команд 

`docker system prune -a --volumes` : очистка диска от Docker-мусора
* `-a`: удаляет все неиспользуемые образы
* `--volumes`: удаляет неиспользуемые тома данных
